# Android 应用图标替换指南

## 概述

本指南提供了两种替换 Android 应用图标的方法：
1. **使用 Android Studio（推荐）** - 最简单、最安全的方法
2. **使用命令行脚本** - 适合喜欢手动控制的开发者

## 方法一：使用 Android Studio（推荐）

### 步骤 1：准备图标文件
- 准备一个高质量的 PNG 图标文件
- 建议尺寸：至少 512x512 像素
- 确保图标是正方形，有透明背景

### 步骤 2：在 Android Studio 中操作
1. 在 Android Studio 中打开项目
2. 在项目视图中，右键点击 `app/src/main/res` 目录
3. 选择 `New` → `Image Asset`
4. 在弹出的对话框中选择 `Launcher Icons (Adaptive and Legacy)`
5. 在 `Foreground Layer` 部分：
   - 点击文件夹图标
   - 选择您的 PNG 图标文件
6. 在 `Background Layer` 部分：
   - 选择背景颜色（或保持透明）
7. 点击 `Next` 预览生成的图标
8. 点击 `Finish` 完成

### 步骤 3：验证结果
- Android Studio 会自动生成所有需要的图标尺寸
- 包括自适应图标和传统图标
- 自动处理不同屏幕密度的适配

### 优势
- ✅ 完全自动化
- ✅ 生成自适应图标
- ✅ 支持所有 Android 版本
- ✅ 无需手动处理尺寸
- ✅ 内置预览功能

## 方法二：使用命令行脚本

### 步骤 1：准备图标文件
1. 将您的图标文件重命名为 `icon.png`
2. 将文件放到项目根目录（`virtual-fitness-coach-android/` 文件夹）

### 步骤 2：运行脚本
```bash
# 给脚本添加执行权限（如果还没有）
chmod +x generate_icons_safe.sh

# 运行图标生成脚本
./generate_icons_safe.sh
```

### 步骤 3：验证构建
```bash
# 测试项目是否能正常构建
./gradlew assembleDebug
```

### 脚本功能说明
- 📁 **自动备份**：将现有图标备份到 `icon_backup` 目录
- 🔄 **生成 WebP 格式**：与项目现有格式保持一致
- 📊 **多尺寸支持**：生成 5 种不同密度的图标
- 🧹 **自动清理**：清理临时文件
- 🔄 **恢复功能**：可以随时恢复原始图标

### 生成的图标尺寸
- **mdpi**: 48x48 px
- **hdpi**: 72x72 px  
- **xhdpi**: 96x96 px
- **xxhdpi**: 144x144 px
- **xxxhdpi**: 192x192 px

### 恢复原始图标
如果需要恢复原始图标：
```bash
cp -r icon_backup/* app/src/main/res/
```

## 图标要求

### 设计建议
- **尺寸**：建议原始图标至少 512x512 像素
- **格式**：PNG 格式，支持透明背景
- **形状**：正方形，内容居中
- **颜色**：确保在不同背景下都清晰可见

### 技术规格
- **文件格式**：PNG（输入）→ WebP（输出）
- **背景**：支持透明背景
- **颜色深度**：24位或32位
- **压缩**：无压缩或轻微压缩

## 常见问题

### Q: 为什么会出现"Duplicate resources"错误？
A: 这通常是因为同时存在 PNG 和 WebP 格式的图标文件。使用我们的安全脚本可以避免这个问题。

### Q: 自适应图标和传统图标有什么区别？
A: 
- **自适应图标**（Android 8.0+）：由前景层和背景层组成，支持动态形状
- **传统图标**：单一图像文件，适用于所有 Android 版本

### Q: 如何测试图标在不同设备上的显示效果？
A: 
1. 使用 Android Studio 的模拟器
2. 在不同分辨率的设备上测试
3. 检查圆形和方形图标的显示效果

### Q: 图标替换后应用没有更新？
A: 
1. 清理项目：`./gradlew clean`
2. 重新构建：`./gradlew assembleDebug`
3. 卸载并重新安装应用

## 最佳实践

1. **备份原始图标**：在替换前总是备份原始文件
2. **测试多种设备**：在不同屏幕密度的设备上测试
3. **保持一致性**：确保图标与应用品牌一致
4. **遵循设计规范**：参考 Android 图标设计指南
5. **版本控制**：将图标文件纳入版本控制
